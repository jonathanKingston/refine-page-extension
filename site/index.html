How refine.page works

Two ways to use it: as a browser extension today, and as a web app workflow too.

Extension workflow

Best for 1-click capture
  1. Capture a page into a self-contained snapshot.
  2. Annotate relevant text and the answer on the frozen page.
  3. Create Q&A labels + evaluation (correctness, “answer in page”, quality).
  4. Review snapshots (approve/decline) with notes.
  5. Export the dataset (and import later).
See extension details

Web app workflow

Great for teams
  1. Start a project and add URLs (or upload HTML snapshots).
  2. Capture on the server (or via a lightweight browser helper).
  3. Label in the browser with the same “Relevant / Answer” tools.
  4. Assign & review work with status and notes.
  5. Export a consistent dataset for training/eval.

FAQ

The basics, straight from the workflow.

What does “inert snapshot” mean?

The captured page is preserved as HTML, and interactive behavior is disabled so labeling stays stable (no navigation, no script reflows).

Where is my data stored?

In the extension, snapshots and labels are stored locally (Chrome storage), and you can export/import as backups.

What can I export?

Snapshots (HTML) plus your annotations, questions, evaluations, and review status — ready to turn into training/eval datasets.

Can this work without the extension?

Yes — the same labeling UI can live on a web page: you’d upload/import snapshots or capture URLs server-side, then annotate in-browser.
